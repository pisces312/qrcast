# 码上通 (QRCast Android)

通过二维码在设备间离线传输小文件的 Android 应用。

扫描由 [QRCast Python](https://github.com/pisces312/qrcast) 生成的二维码，即可在手机上接收文件。无需网络、无需配对，即扫即得。

An Android app for offline file transfer via QR codes. Scan QR codes generated by [QRCast Python](https://github.com/pisces312/qrcast) to receive files on your phone. No network, no pairing — just scan.

---

## 功能 / Features

- **分块传输模式**：支持大文件分多个二维码传输，自动组装还原
- **原始数据模式**：单张二维码直接编码文件内容，适合小文件
- **相机扫码**：实时摄像头扫描，支持自动对焦
- **图库选图**：从相册选择二维码图片解析
- **可配置输出目录**：设置页面可修改文件保存位置
- **时间戳文件名**：接收文件自动命名为 `yyyyMMdd_HHmmss_SSS` 格式
- **调试日志**：内置日志查看器，方便排查问题
- **打开/分享**：接收完成后可直接打开或分享文件

## 协议 / Protocol

### 分块模式 (CHUNKED)

每个二维码编码一个数据块，格式：

Each QR code encodes one data chunk:

```
[4 bytes: seq] [4 bytes: total] [remaining: payload]
```

- 序号从 0 开始 / Sequence starts from 0
- 接收端收集所有块后按序号拼接，自动识别 7z/zip 压缩包并解压
- Receiver collects all chunks, assembles by sequence, auto-detects & decompresses 7z/zip

### 原始数据模式 (RAW)

整个二维码直接编码文件原始字节，无分块头。适合不超过单张二维码容量的文件（QR version 40 约 2953 字节）。

The entire QR encodes raw file bytes with no chunking header. Suitable for files within a single QR's capacity (~2953 bytes at QR version 40).

## 构建 / Build

要求 / Requirements:
- Android SDK (compileSdk 35)
- JDK 17
- Gradle 9.4.1

```bash
./gradlew assembleDebug
```

APK 输出 / Output: `app/build/outputs/apk/debug/app-debug.apk`

## 技术栈 / Tech Stack

- Kotlin + AndroidX
- CameraX (camera preview & image analysis)
- ML Kit Barcode Scanning (QR code recognition)
- SharedPreferences (settings persistence)
- FileProvider (file sharing)
